> 状态: 已确认

## 概述

FerrumDB V0.1 是项目的基础架构版本，目标是搭建可工作的项目骨架，验证技术栈可行性，为后续 MVP 开发奠定基础。

---

## 核心目标

1. **搭建项目骨架** - 完成 Tauri + React + TypeScript 项目初始化
2. **验证数据库连接** - 实现 MySQL 和 PostgreSQL 的基础连接能力
3. **本地配置存储** - 使用 SQLite 存储连接配置
4. **UI 基础布局** - 实现主界面布局和连接管理界面

---

## 技术选型

| 层级 | 技术 | 理由 |
|------|------|------|
| 桌面框架 | Tauri 2.x | 轻量、安全、原生性能 |
| 前端框架 | React 18 + TypeScript | 生态成熟、类型安全 |

| 本地存储 | SQLite (sqlx) | 轻量、嵌入式、与数据库驱动统一 API |

## 技术细节

### sqlx 编译时验证

sqlx 默认在编译时验证 SQL 语句，需要 `DATABASE_URL` 环境变量。项目采用以下策略：

- **开发环境**：使用 `.env` 文件配置本地数据库连接
- **CI/CD 环境**：使用 sqlx offline 模式，预先生成 `.sqlx` 查询元数据文件
- **测试**：使用 Docker Compose 启动临时数据库实例

### 测试数据库策略

V0.1 的数据库连接测试采用以下方案：

- 使用 Docker Compose 定义测试用的 MySQL 和 PostgreSQL 容器
- CI 流程中自动启动/销毁容器
- 本地开发可选择性启动测试数据库
- 如无测试数据库，跳过集成测试（使用 `#[ignore]` 标记）

### F2: 连接管理界面

**目标**: 用户可以管理数据库连接配置

**UI 参考**: [`stitch/connections_dashboard/code.html`](../../stitch/connections_dashboard/code.html)

**功能点**:
- [ ] **侧边栏导航** - 显示主导航菜单
- [ ] **顶部工具栏** - 包含搜索框和"添加连接"按钮
- [ ] **连接卡片网格** - 展示已保存的连接配置
- [ ] **添加连接弹窗** - 表单输入连接信息
- [ ] **编辑连接** - 修改已有连接配置
- [ ] **删除连接** - 移除连接配置（需确认）

**连接配置字段**:

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | string | 自动 | 唯一标识 |
| name | string | 是 | 连接名称 |
| type | enum | 是 | mysql / postgresql |
| host | string | 是 | 主机地址 |
| port | number | 是 | 端口号 |
| username | string | 是 | 用户名 |
| password | string | 是 | 密码（加密存储） |
| database | string | 否 | 默认数据库 |
| environment | enum | 否 | production / staging / development |
| tags | string[] | 否 | 自定义标签 |
| created_at | datetime | 自动 | 创建时间 |
| updated_at | datetime | 自动 | 更新时间 |

**注意**: V0.1 阶段不实现实际数据库连接功能，仅管理配置数据。

**交互状态规范**:

| 状态 | 场景 | UI 表现 |
|------|------|----------|
| **Empty** | 无保存的连接配置 | 显示空状态插图 + "添加第一个连接" 按钮 |
| **Loading** | 加载/保存配置时 | 卡片区域显示骨架屏，按钮显示 loading spinner |
| **Error** | 操作失败 | Toast 通知 + 错误信息，保留用户输入 |
| **Success** | 操作成功 | 短暂 Toast 确认，自动关闭弹窗 |
| **Confirm** | 删除操作 | 模态确认框，要求用户确认 |

**搜索/过滤状态**:
- 输入搜索词时实时过滤卡片列表
- 无匹配结果时显示 "无匹配连接" 提示

### F3: 本地存储

**目标**: 使用 SQLite 持久化连接配置

**实现方式**:
- 应用启动时检查/创建数据库文件
- 连接配置表自动迁移
- 密码字段加密存储：
  - 算法：AES-256-GCM
  - 密钥派生：从操作系统密钥库获取（Windows DPAPI / macOS Keychain / Linux Secret Service）
  - 备选方案：使用 PBKDF2 从用户主密码派生（如不支持系统密钥库）

**错误处理策略**:

| 错误类型 | 处理方式 |
|----------|----------|
| 目录创建失败（权限） | 显示错误对话框，提供选择其他存储位置的选项 |
| 数据库文件损坏 | 提示用户备份并重建数据库，或尝试自动恢复 |
| 迁移失败 | 回滚到上一版本，记录错误日志，提示用户更新应用 |
| 密钥获取失败 | 降级到内存模式（不持久化），提示用户检查系统密钥库 |

**存储位置**:
- Windows: `%APPDATA%/ferrum-db/config.db`
- macOS: `~/Library/Application Support/ferrum-db/config.db`
- Linux: `~/.config/ferrum-db/config.db`

### F4: 数据库连接层

**目标**: 验证 MySQL 和 PostgreSQL 连接能力

**实现范围**:
- [ ] 定义统一的异步数据库连接 trait（async）
- [ ] 实现 MySQL 连接器（使用 sqlx）
- [ ] 实现 PostgreSQL 连接器（使用 sqlx）
- [ ] 连接测试命令（供 V1.0 使用）

**注意**: V0.1 仅验证连接层可用性，通过单元测试验证连接能力。该层代码在 V0.1 阶段不暴露给前端使用，但必须通过测试确保功能正确。

## 非功能需求

### NFR1: 性能

- 冷启动时间 < 3 秒（首次运行除外）
- 空闲内存占用 < 150MB

### NFR2: 安全

- 密码不以明文存储
- 配置文件权限受限（仅当前用户可读）

### NFR3: 兼容性

- Windows 10/11
- macOS 11+ (Intel & Apple Silicon)
- Ubuntu 20.04+
