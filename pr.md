这份需求文档旨在为**开发人员**量身打造一款高效、轻量的数据库可视化管理工具）。考虑到开发者的工作流，重点将放在**性能、快捷键支持、多协议兼容**以及**现代化的AI辅助**上。

---

## 1. 项目概述
### 1.1 目标
创建一个跨平台的桌面客户端，解决现有工具（如 Navicat 过于沉重、DBeaver UI 繁琐）的痛点。
### 1.2 核心价值
* **开发者优先**：原生支持 SSH 隧道、自定义脚本和代码级操作。
* **极致速度**：即便处理百万级数据，UI 也不应卡顿。
* **现代化**：深度集成 AI SQL 助手与 ER 图自动生成。

---

## 2. 目标用户画像
* **后端工程师**：需要频繁调试 SQL，查看表结构。
* **DBA/运维**：监控数据库状态，处理数据迁移。
* **全栈/前端**：需要直观地修改测试数据，无需记忆复杂的命令。

---

## 3. 功能需求

### 3.1 连接与安全管理
| 功能模块 | 需求描述 |
| :--- | :--- |
| **多引擎支持** | 必须支持 PostgreSQL, MySQL, SQLite, MongoDB, Redis, ClickHouse。 |
| **连接管理** | 支持分组（生产、预发、开发）、标签颜色区分。 |
| **SSH 隧道** | 内置 SSH 隧道支持，支持 Key Pair 及 Jump Server（跳板机）。 |
| **环境隔离** | 生产环境操作需强制二次确认，或开启“只读模式”。 |

### 3.2 智能 SQL 编辑器
* **LSP 集成**：支持智能补全（表名、字段名、关键字）、语法高亮。
* **AI 助手**：集成 LLM（如 GPT-4 或本地模型），支持自然语言转 SQL，解释复杂的查询计划。
* **多任务执行**：支持选中部分代码执行，结果集分标签页展示。
* **Explain 可视化**：将 `EXPLAIN ANALYZE` 的结果转化为逻辑树图，定位查询瓶颈。



### 3.3 数据操作与可视化
* **电子表格视图**：支持类似 Excel 的单元格编辑、批量删除、排序和高级过滤。
* **ER 图自动生成**：一键生成数据库关系图，支持拖拽布局，导出为 SVG/PDF。
* **JSON 支持**：针对 JSONB 字段提供专门的层级编辑器，而非简单的文本框。

### 3.4 结构设计
* **无损建模**：支持图形化修改表结构，并自动生成对应的 `ALTER TABLE` 语句供用户预览，而非直接提交。
* **版本控制同步**：支持将表结构变化导出为迁移脚本（Migration files），兼容 Golang (Golang-migrate) 或 Python (Alembic)。

---

## 4. 非功能需求

### 4.1 性能要求
* **冷启动时间**：在 macOS (Apple Silicon) 上应小于 2 秒。
* **内存占用**：基础运行内存控制在 200MB 以内（拒绝 Electron 式的资源暴食）。
* **滚动体验**：支持虚拟列表，百万级数据流畅滚动。

### 4.2 交互体验 (UX)
* **全键盘操作**：提供类似 VS Code 的命令面板 (`Cmd + Shift + P`)，所有功能均可快速检索。
* **深色模式**：完美的适配，尤其是代码区域的对比度。
* **插件系统**：提供 API 允许用户使用 **Rust** 或 **TypeScript** 编写自定义数据模拟器或导出插件。

---

## 5. 建议技术栈 (针对 2026 开发趋势)
由于你是 Mac 用户且熟悉 Rust/Go，建议采用以下方案以保证极致性能：

* **前端/UI 层**：使用 **Rust + Tauri**。相比 Electron，Tauri 产生的包体更小，性能更接近原生，且安全性更高。
* **后端逻辑**：Rust (使用 `sqlx` 或 `diesel` 处理多数据库连接)。
* **数据存储**：使用 SQLite 存储用户的本地连接配置和查询历史。
