# FerrumDB 版本计划

> 基于 pr.md 需求文档制定的开发路线图

---

## V0.1 - 项目基础架构 (2-3周)

**目标**: 搭建项目骨架，验证技术栈可行性

| 模块 | 功能点 |
|------|--------|
| 项目初始化 | Tauri + Rust 项目结构，前端框架选型 |
| 数据库连接层 | `sqlx` 集成，MySQL/PostgreSQL 基础连接 |
| 本地存储 | SQLite 存储连接配置 |
| UI 框架 | 基础布局、深色模式支持 |

---

## V1.0 - MVP 核心功能 (6-8周)

**目标**: 可用的数据库管理工具，覆盖 80% 日常场景

| 模块 | 功能点 |
|------|--------|
| 连接管理 | 分组/标签、连接池、断线重连 |
| SSH 隧道 | Key Pair 认证、Jump Server 支持 |
| SQL 编辑器 | 语法高亮、基础补全、选中执行、多标签结果 |
| 数据视图 | 虚拟列表（百万级）、单元格编辑、排序过滤 |
| 安全 | 生产环境二次确认、只读模式 |
| 性能 | 冷启动 < 2s、内存 < 200MB |

**支持的数据库**: MySQL, PostgreSQL

---

## V1.1 - 智能化增强 (4-5周)

**目标**: AI 辅助 + 性能分析

| 模块 | 功能点 |
|------|--------|
| AI SQL 助手 | 自然语言转 SQL、查询解释 |
| Explain 可视化 | 执行计划树图、瓶颈定位 |
| LSP 增强 | 表名/字段名智能补全、语法检查 |
| JSON 编辑器 | JSONB 层级编辑器 |

---

## V1.2 - 数据库扩展 (3-4周)

**目标**: 支持更多数据库类型

| 模块 | 功能点 |
|------|--------|
| SQLite 支持 | 本地数据库管理 |
| Redis 支持 | 键值管理、实时监控仪表盘 |
| MongoDB 支持 | 文档浏览、聚合查询 |
| ER 图生成 | 一键生成、拖拽布局、导出 SVG/PDF |

---

## V1.3 - 结构设计 (3-4周)

**目标**: 图形化建模能力

| 模块 | 功能点 |
|------|--------|
| 表结构编辑 | 图形化修改、预览 ALTER 语句 |
| 迁移脚本 | 导出 Migration 文件 |
| 版本控制同步 | 支持 Alembic 等框架格式 |

---

## V2.0 - 协作与生态 (6-8周)

**目标**: 团队协作 + 插件系统

| 模块 | 功能点 |
|------|--------|
| 云端同步 | 加密共享连接配置 |
| 插件系统 | Rust/TypeScript API |
| ClickHouse 支持 | 大数据分析场景 |
| 监控仪表盘 | 连接池状态、慢查询日志 |

---

## 里程碑时间线

```
V0.1 ── V1.0 ────── V1.1 ──── V1.2 ──── V1.3 ──── V2.0 ──────
 │       │           │         │         │         │
架构    MVP         AI增强    多DB      建模      协作
(3周)  (8周)       (5周)    (4周)     (4周)     (8周)
```

**总预估**: 约 32 周 (8个月)

---

## 技术栈确认

- **前端/UI 层**: Rust + Tauri
- **后端逻辑**: Rust (sqlx / diesel)
- **本地存储**: SQLite
- **AI 集成**: Claude API / 本地模型

---

## 风险与依赖

| 风险项 | 应对策略 |
|--------|----------|
| Tauri 学习曲线 | 提前调研示例项目 |
| 多数据库协议差异 | 抽象统一连接层 |
| AI API 成本 | 支持本地模型备选方案 |
| 虚拟列表性能 | 参考成熟组件库实现 |
